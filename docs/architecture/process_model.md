# Process Model

## Overview

A Process is the fundamental execution unit within VOS. Every running application, system service, and background task executes as a VOS Process.

Although VOS runs as a single application on the host operating system, it maintains its own virtual process table, process identifiers (PIDs), process states, and scheduling policies.

This allows VOS to behave like a real operating system while remaining platform-independent.

# Objectives

The Process Model is designed to:

* Represent every running task as a Process.
* Provide process isolation within the VOS runtime.
* Enable scheduling and lifecycle management.
* Track resource usage.
* Support future multitasking features.

# Process Structure

Each process contains:

* Process ID (PID)
* Name
* Owner
* Status
* Priority
* Start Time
* Thread Handle
* Memory Statistics
* CPU Statistics
* Application Instance

# Process States

Processes may exist in one of the following states:

* Created
* Ready
* Running
* Sleeping
* Waiting
* Suspended
* Terminated

State transitions are controlled by the Runtime and Scheduler.

# Process IDs

Every process receives a unique PID generated by the Runtime.

PIDs are never reused during the same runtime session.

# Resource Tracking

Each process tracks:

* CPU usage
* Memory usage
* Runtime duration
* Event statistics

Resource information represents the Runtime's view of the process while using the host operating system's actual CPU and RAM.

# Process Communication

Processes communicate using:

* Event Bus
* Runtime Services
* Shared Runtime APIs

Direct communication between applications should be avoided.

# Future Features

Future versions may support:

* Process priorities
* Parent/child processes
* Background services
* Process groups
* Inter-process communication (IPC)
* Crash recovery

# Summary

Every executable component in VOS exists as a Process managed by the Runtime. The Process Model provides the foundation for multitasking, scheduling, application management, and resource monitoring.
